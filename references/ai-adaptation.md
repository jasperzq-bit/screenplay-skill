# AI生成适配原则

这份文件说明在为AI生成视频创作剧本时，必须注意的技术边界和创作策略。
写任何场景描述前必读。

---

## 一、AI生成视频的核心技术限制

了解这些限制，才能在创作时主动规避，而不是事后修改：

| 限制类型 | 具体表现 | 应对策略 |
|---|---|---|
| 人物一致性漂移 | 同一人物在不同场景里可能面貌不同 | 每个人物形象描述必须精确，关键特征固化 |
| 复杂动作失控 | 肢体打斗、奔跑、多人互动容易变形 | 尽量用静态情绪场景替代动作场景 |
| 群体场景混乱 | 超过4个人物的场景容易出现错误 | 核心对话控制在2–3人 |
| 场景一致性 | 同一地点多次出现时布景可能不同 | 固化场景描述，每次引用同一版本 |
| 对话口型问题 | AI生成的人物说话口型可能不准 | 台词节奏设计时考虑口型自然衔接 |
| 长时间连续镜头 | 长镜头生成质量不稳定 | 单个场景时长控制，多用剪切 |

---

## 二、场景设计原则

### 2.1 空间复杂度控制

给每个场景标注 AI复杂度 等级（写在场景描述里）：

**简单**：室内、2人以内、静态情绪、固定光源
- 示例：出租屋，林昊一个人，晨光从窗帘缝透进来

**中等**：室外或较大室内、2–3人、轻微动作、自然光
- 示例：学校门口，傍晚，3人站着说话

**复杂**：群体场景/大型空间/复杂动作/特殊光效
- 示例：宴会厅，十几个宾客，多个区域同时发生的事

**创作规则**：
- 简单场景：随意使用
- 中等场景：主要场景可用，但限制人物动作幅度
- 复杂场景：每集最多1–2个，且必须是剧情关键节点才用

### 2.2 优先使用的场景类型

这些场景在AI生成时效果最稳定：

✅ 室内封闭空间（公寓、办公室、咖啡馆角落、电梯、走廊）
✅ 两个人正面或侧面对话
✅ 人物独处的内心场景
✅ 窗边/门口等有自然光线的半开放空间
✅ 简单的夜晚室外（两个人，路灯或店铺光）

### 2.3 尽量避免的场景类型

❌ 大型宴会/婚礼/发布会（超过5人同框）
❌ 复杂打斗或追逐（用结果替代过程）
❌ 快速移动的交通工具内景
❌ 特殊天气（暴雨、大雪、雾）与复杂情节结合
❌ 同一场景内多次场景切换

**变通方法**：
- 需要群体场景 → 用"会场外的走廊"替代"会场内"，只保留2–3个核心人物
- 需要打斗 → 写结果（人倒地/门被关上/血迹出现），不写过程
- 需要大场面 → 用声音+反应替代全景展示（听到远处爆炸声，主角的脸）

## 三、人物视觉一致性管理

### 3.1 人物外形"固定资产"

每个核心人物的以下特征，一旦在人物设定里确定，全片不可改变：

- **面部骨架**（脸型、眼型、鼻型）
- **标志性细节**（疤痕、特征性表情、某个习惯动作）
- **基础发型方向**（长短、颜色、质感方向）

同一人物在不同场景的着装可以变化，但以上三项永远固定。

### 3.2 场景首次出现时的人物描述

每当一个重要场景首次出现人物，在动作描述里加入简短的外形参照（方便生图时调取）：

```
（林昊靠着窗边站着——十八岁的脸，深眼窝，习惯性地把手插在兜里，
  这个姿势让他看起来比实际年龄稳。）
```

不需要每次都写完整，但要有锚点细节。

## 四、场景描述的生图可用性检查

写完每个场景描述后，问自己：

1. **定位清楚吗？** 生图AI能知道这是什么地方吗？
2. **光线明确吗？** 主光源是什么，从哪个方向来？
3. **人数清楚吗？** 场景里有几个人，站在什么位置？
4. **情绪氛围清楚吗？** 这个画面应该给人什么感觉？
5. **有无模糊地带？** 有没有可以被AI随机发挥的信息漏洞？

漏洞越少，生成结果越稳定。

## 五、固化场景系统

### 5.1 什么是固化场景

剧情中多次出现的地点（主角家、反派办公室、两人常去的咖啡馆），必须建立"固化场景描述"，每次出现时引用同一版本描述，不重写、不即兴。

### 5.2 固化场景描述规范

在人物关系图谱之后、分集大纲之前，单独列出所有固化场景：

```
## 固化场景库

**林昊的出租屋**
东南朝向的老旧单间，约15平米。墙皮在角落有些剥落，窗帘是廉价的米白色。
床靠东墙，床头柜是一个生锈的铁架，上面总放着一个翻盖手机和几本旧书。
地板是深色木纹的劣质地板革。
光线：白天有一道阳光斜进来，晚上只有床头的白炽灯。
氛围关键词：拮据但干净，有一种被生活压着却没有垮的感觉。

**林家宴会厅**
（下次写到这个场景时引用并补充……）
```

每个固化场景描述不超过150字，精准、可复现。

## 六、短剧AI生成的节奏建议

每集时长1.5–3分钟，对应生成视频的镜头数量：

| 时长 | 建议场景数 | 建议镜头数 |
|---|---|---|
| 1.5分钟 | 3–4个场景 | 15–25个镜头 |
| 2分钟 | 4–5个场景 | 20–35个镜头 |
| 3分钟 | 5–7个场景 | 30–50个镜头 |

每个场景尽量不超过8–10个镜头切换，让生成工具有足够的时间维持一致性。
