# 多集竖屏短剧 · 完整输出格式规范

## 输出模块顺序（固定，不可省略任何一项）

```
01. 剧本基础信息表
02. 故事梗概
03. 核心冲突线 + 情绪曲线设计
04. 人物设定（每个人物完整五板块）
05. 人物关系图谱
06. 分集大纲
07. 分集剧本
```

---

## 01. 剧本基础信息表

一张表，开门见山，固定字段，全部必填：

| 字段 | 内容 |
|---|---|
| 剧名 | |
| 题材 | 如：重生爽剧 / 豪门虐恋 / 替身打脸 / 都市逆袭 |
| 总集数 | |
| 单集时长 | 建议1.5–3分钟 |
| 目标受众 | 如：18–35岁女性，下班解压用户 |
| 核心卖点 | 一句话，让人想点进去的那句话 |
| 情绪定位 | 如：先虐后甜 / 打脸爽感 / 虐渣逆袭 / 甜宠日常 |
| 平台适配 | 抖音 / 快手 / 微短剧APP / 海外竖屏 |
| AI生成备注 | 场景复杂度控制要求，人物数量上限 |

---

## 02. 故事梗概

**两个层次，缺一不可：**

**一句话梗概**：类似海报上那句话，包含主角 + 核心困境 + 情绪温度。15–30字，让陌生人秒懂这是什么类型、有没有他要的爽点。

**完整梗概**：800–1500字。必须包含：
- 世界观/背景设定（何时何地，这个世界的规则是什么）
- 主角的起点状态和核心动机
- 反派/对立方的逻辑（不能只是坏，要有自己的动机）
- 核心冲突如何建立
- 情感线如何推进
- 高潮是什么
- 结局方向（不需要剧透细节，但要给出走向）

追求可读性，不追求文学性。任何人读完能在脑海里放完这部剧。

---

## 02.5 核心事件真相表

**必须在人物设定之前完成。未完成此表，不得进入人物设定环节。**

这张表的作用是把故事里所有「幕后发生的关键事件」显式拆解，
强制验证每一个事件的执行者、动机、信息掌握程度和知情范围。
任何在梗概里被轻描淡写带过的事件，在这里都必须逐条交代清楚。

**「关键事件」的定义：**
- 推动整个故事主线的幕后行为（如调包、陷害、收买、谋杀）
- 在剧情中已发生、但观众稍后才知道的事件
- 涉及多方人物且需要各方动机支撑的事件
- 任何「如果这件事没发生，故事就不存在」的事件

**填写格式（每个关键事件一行，不可合并）：**

| # | 事件描述（具体，不可模糊） | 执行者 | 执行者动机（必须具体到个人处境） | 执行者当时掌握的信息 | 知情者 / 不知情者 |
|---|---|---|---|---|---|
| 01 | | | | | |
| 02 | | | | | |

**每填完一行，必须回答以下三个问题。全部回答「是」才可继续填下一行：**

问题一：这个执行者有合理动机做这件事吗？
动机必须具体到这个人的处境、欲望或恐惧，不能写「为了利益」「因为坏」这类
抽象描述。抽象动机等于没有动机。

问题二：这个执行者当时掌握的信息，足以让他做出这个判断和决定吗？
信息不足的人无法做出精准的行动。如果执行者需要某条信息才能行动，
必须说明他从哪里、在什么时间获得了这条信息。

问题三：这个事件和其他已填事件之间，有没有时间线或因果链的矛盾？
同一件事不能同时有两个互相矛盾的描述。

**常见逻辑硬伤对照清单（填表前先扫一遍）：**

- 执行者的行动动机和其性格/处境存在根本矛盾
  示例：一个刚刚满足了「想要孩子」愿望的人，同时又因为「太想要孩子」
  而接受另一个婴儿——两个动机互相消解，必须选一个或重新设计

- 某个关键事件没有明确的执行者，只有结果
  示例：「婴儿被调换了」——谁调换的？为什么？怎么做到的？
  没有执行者的事件是叙事漏洞，不是悬念

- 受害者对某件显而易见的事毫无反应，但故事没有解释为什么
  示例：医院里突然多出来一个陌生婴儿，产妇毫无疑问地接受了
  ——必须给出这个「毫无疑问」的成立条件

- 两个人物都声称对某件事不知情，但这件事需要有人主动执行

- 某人做了一件需要特定资源或渠道的事，但小传里没有说明他/她有这个资源

**此表通过验证后，将成为后续人物小传和剧本写作的基准文件。
人物小传里涉及的所有关键行为，必须能在此表里找到对应记录。**

## 03. 核心冲突线 + 情绪曲线设计

**冲突线**：3–4条并行主线，每条一句话说清楚"谁 vs 谁，因为什么"。

示例格式：
- 身份冲突线：[真实身份] vs [被压制/顶替的处境]
- 情感冲突线：[真情] vs [误会/契约/猜忌]
- 复仇冲突线：[受害者的反击] vs [施害者的掌控]
- 揭秘线：[隐藏真相] vs [逐渐曝光的节奏]

**情绪曲线**：按集数段标注主情绪。格式：

```
第1–X集：[情绪类型]（[核心事件类型]）→
第X–X集：[情绪类型]（[核心事件类型]）→
...
最后X集：[大结局情绪]（[收尾方式]）
```

情绪类型只用这几个词：虐 / 甜 / 爽 / 悬念 / 催泪 / 震撼

这条曲线是每集创作时的情绪红线，每集情绪不能偏离所属段的主情绪定位。

## 04. 人物设定

每个人物必须包含以下五个板块，全部必填。
详细写作标准见 `references/character-standards.md`。

```
A. 基础档案（姓名、年龄、身份定位、故事功能）
B. 人物小传（≥200字，背景+经历+动机+与主线的关系起点）
C. 详细形象描述（≥150字，供AI生图使用，含外形+气质+标志细节+着装+情绪变化）
D. 性格特点（3–5个核心词，每个词附具体行为体现）
E. 台词特点（口头习惯+情绪变化下的台词风格+示范台词）
```

主要人物（主角、反派、女主）完整写。次要人物至少写A+B+C+E。

---

## 05. 人物关系图谱

结构化展示所有人物之间的关系，保证任何新加入的创作者5分钟内看懂所有人的站位。

必须包含：
- 每对人物之间的关系类型（血缘 / 爱情 / 契约 / 敌对 / 同盟 / 联手）
- 关系的起点和走向（如：从契约→误解→真情→破裂→重建）
- 隐藏关系（剧情里还未揭露的）单独标注⚠️，提醒需要提前埋线

格式：文字图或层级列表，清晰即可，不必追求美观。

---

## 06. 分集大纲

每集一行，表格格式，五列：

| 集数 | 标题 | 核心事件 | 情绪点 | 本集钩子 |
|---|---|---|---|---|

要求：
- 核心事件必须具体（不写"两人产生矛盾"，要写"沈念在宴会上被当众羞辱，陆景琛冷眼旁观"）
- 情绪点精准：爽 / 虐 / 悬念 / 甜 / 催泪
- 本集钩子：集尾那个让人必须点下一集的悬念，一句话描述

---

## 07. 分集剧本

### 每集内部固定结构：

**集标题页**
```
第X集·《集标题》
本集核心情绪：[情绪类型]
本集钩子：[一句话描述集尾悬念]
```

**场景块**（每场戏一个场景块，包含以下四部分）：

```
【场景编号·地点·时间·内/外景】

（场景视觉描述：空间大小、光线、主要陈设、氛围关键词。
  必须具体到可以生图的程度。AI复杂度：简单/中等/复杂。）

【运镜备注：开场景别→推进方式→情绪节点的镜头处理】

人物动作和肢体描述（带画面感，一个动作一句话）

人物名（情绪/动作说明）
台词内容
```

### 短剧节奏强制要求：

- **前15秒钩子**：第一场戏必须是冲突爆发 / 信息炸弹 / 视觉最强一刻，不能从交代背景开始
- **30–45秒节点**：每隔30–45秒必须有一个情绪高点
- **集尾悬念**：最后一场戏不能是收口，必须是一个让人焦虑/好奇/心痛的停留点
- **台词密度**：废话率为零，每句台词都在推进情节或情绪

### 第一场戏开场方式优先级（高→低）：

1. 最激烈的冲突场面（前世死亡 / 当众打脸 / 关键秘密揭露）
2. 最大信息量的一刻（揭示核心矛盾）
3. 最强的视觉画面（能让人停下滑动手指的那一帧）
4. 反转性开场（观众以为是A，其实是B）

❌ 绝对不从"交代背景"开场
❌ 绝对不从"平静日常"开场
